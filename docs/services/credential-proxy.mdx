## Purpose

An HTTP sidecar that intercepts outbound API requests, resolves credentials from Doppler, and injects them — the agent process never sees raw tokens. Routing is resource-driven: each resource (e.g. `github-pat`, `youtube-hay-maker`) declares which hostnames it covers and how credentials are injected.

## Architecture

```
agent-youtube → fetch('https://youtube.googleapis.com/...') → [interceptor] → YouTube API
                                                                    ↑
                                                       credential resolved from Doppler
                                                       + injected as Authorization: Bearer
```

The proxy is invisible to `agent-youtube`. Both work identically without the other.

## Routing

Credentials are resolved in priority order:

1. **X-Resource header** — explicit resource name, e.g. `X-Resource: github-pat`
2. **Hostname fallback** — proxy scans all resources and matches on `config.urls`

The DB is the authoritative source. Resolution follows `resource_links` with `link_type='credential'` to find the credential spec for a given identity resource. An in-memory fallback table is used when the DB is unavailable. Response header `x-resolution-source` indicates whether credentials were resolved via `db` or `fallback`.

## Supported Resources

| Resource | Hostnames | Inject Mode | Egress Proxy |
|---|---|---|---|
| `github-pat` | `api.github.com`, `github.com` | Bearer | — |
| `youtube-hay-maker` | `youtube.googleapis.com`, `googleapis.com` | Bearer | — |
| `supabase-<your-project-ref>` | `*.supabase.co` | Supabase (Authorization + apikey) | — |
| `instagram-trianglebender` | `graph.facebook.com`, `graph.instagram.com` | Bearer | `PROXY_02_URL` |
| `instagram-squarebent` | `graph.facebook.com`, `graph.instagram.com`, `api.ipify.org` | Bearer | `PROXY_01_URL` |

## Egress IP Routing (SOCKS5 Proxy)

Resources can declare a `proxyResourceId` in their DB config to route outbound requests through a dedicated SOCKS5 proxy. When set:

1. The proxy resolves the `urlEnvVar` from the proxy resource's config
2. Fetches the SOCKS5 URL from Doppler
3. Routes the outbound request via `SocksProxyAgent` + `node:https.request`

If a proxy is configured but the URL cannot be resolved, the proxy returns **502** with `{ error: "Proxy URL not resolved" }`. Direct (non-proxied) requests continue to use native `fetch`.

## Process Attribution (EVAL_PROCESS_ID)

When `EVAL_PROCESS_ID` is set to a valid UUID in the environment, the fetch interceptor adds `X-Process-ID: <uuid>` to every proxied request. The proxy uses this to attribute credential usage back to the originating eval process in emitted agent events.

To wire this automatically across all child Node.js processes, `launch.ts` sets:

```bash
export EVAL_PROCESS_ID=<campaign-id>
export NODE_OPTIONS="--import /path/to/interceptor-boot.mjs"
```

`interceptor-boot.mjs` loads the TypeScript interceptor via `tsx` and installs the global fetch patch. If loading fails, it logs a warning and allows the process to continue.

Security: `X-Process-ID` is validated as a UUID before being set (invalid values are dropped) and is stripped from outbound headers before requests are forwarded to upstream third-party servers.

## Security Features

| Feature | Mechanism |
|---|---|
| Destination validation | Credentials only injected when hostname is in resource's `urls` allowlist |
| X-Resource sanitization | Strict `[a-z0-9\-_]{1,128}` allowlist; rejects CRLF and path-traversal |
| SSRF guard | DNS-resolves hostnames and blocks RFC 1918, loopback, link-local, CGNAT, and IPv4-mapped IPv6 ranges |
| Uniform error responses | All lookup failures return no credentials — no enumeration oracle |
| Doppler scope constraint | `allowedKeys` set enforced in `DopplerCredentialStore`; closes confused-deputy attack |
| No redirect following | Proxied requests never follow redirects |
| X-Process-ID stripping | Internal process UUID stripped before forwarding to upstream servers |
| AI provider bypass | Requests to known AI API hosts (Anthropic, OpenAI, Gemini, Mistral, Together, OpenRouter) are never rerouted — they carry their own auth. Bypass URLs from `ANTHROPIC_BASE_URL` / `CREDENTIAL_PROXY_BYPASS_URLS` are boundary-checked to prevent subdomain spoofing. |

## Configuration

| Env Var | Default | Description |
|---|---|---|
| `CREDENTIAL_PROXY_PORT` | `7447` | Port the sidecar listens on |
| `DOPPLER_SERVICE_TOKEN` | — | Doppler service token for secret fetching (preferred) |
| `DOPPLER_TOKEN` | — | Fallback Doppler token (used when `DOPPLER_SERVICE_TOKEN` is not set) |
| `DOPPLER_PROJECT` | — | Doppler project (set to your project name) |
| `DOPPLER_CONFIG` | — | Doppler config / environment (e.g. `dev`, `prd`) |
| `DATABASE_URL` | — | Postgres URL for resource_links table (optional; falls back to in-memory). When set, also enables structured log persistence via `DatabaseDrain`. |
| `LOG_LEVEL` | `debug` (dev) / `info` (prod) | Minimum log level to emit (`debug`, `info`, `warn`, `error`, `fatal`). |
| `ANTHROPIC_BASE_URL` | — | Custom Anthropic gateway URL (e.g. `http://172.18.0.1:8317`). Requests to this base URL bypass the proxy. |
| `CREDENTIAL_PROXY_BYPASS_URLS` | — | Comma-separated list of base URLs to bypass the proxy (e.g. `https://my-gateway.internal`). Use for non-standard AI endpoints not in the built-in bypass list. |

## Token Exchange (YouTube)

YouTube uses OAuth refresh tokens. The `exchangeRefreshToken` function in `packages/agent-youtube` performs the OAuth exchange using `YOUTUBE_REALENTRY_CLIENT_ID` and `YOUTUBE_REALENTRY_CLIENT_SECRET`. It calls Google's token endpoint directly via the native (unpatched) `fetch`, bypassing the proxy interceptor.
